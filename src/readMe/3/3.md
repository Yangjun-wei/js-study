##简答题
```javascript
1.代码行数较少的小型程序，程序入口清楚、代码逻辑短、分支少，
程序结构和执行流程肉眼扫几遍基本一目了然，不论是写还是改很快就能完成。
但是对于代码行数上万、几万几十万几百万的大型程序，无论什么样的大脑，
不论把代码从头到尾看几遍，都不可能对整个程序从头到尾的执行流程清清楚楚，
代码量大也必须让多个人来完成而不是一个人去写。每个人都只能了解自己写的
代码的逻辑，所以必须对程序不同的功能做解耦的切割，每个相对独立的功能
称之为模块。除了方便多人协作开发，从项目管理的角度也必须把程序的不同功能
分割成相对独立的模块，顶级项目管理人员只需知道各个模块的功能是什么，至于各个模块
内部是怎么实现的，这个顶级项目管理人员不用管，可以交给次一级的项目管理人员负责。
这个就是程序项目的工程化。不论是人类社会的政府，还是航空航天，还是任何工业项目，项目结构
其实都是按模块划分的金字塔结构。还有通用的功能可以抽成模块复用。
工程化能解决的问题是:
(1).解决大型项目难于难于管理的问题；
(2).程序模块化划分，减轻程序员编写功能时的心智负担。通用功能和底层操作
直接调用已有模块；
(3).提高生产效率。开发、测试、编译打包、部署全部自动化，无需每次手工进行。
2.硬性规定项目目录结构、开发范式、工具配置、基础代码。简单说，
就是把项目工程的创建标准化、流水线化、制度化。
```

##grunt编译项目  编译的项目是pages-boilerplate
```javascript
1.grunt默认使用项目根目录下的gruntFile.js文件作为配置文件;gruntFile.js
文件需要导出一个以grunt对象为唯一参数的函数供grunt库调用;
2.导出的函数的参数grunt对象提供initConfig方法供我们预定义一些数据,
通过registerTask定义自定义的任务,自定义任务中可以通过this引用到预定义的数据;
3.sass、高版本js、react等需要用专用的第三方库编译。常用的有grunt-sass、grunt-babel等。
第三方库的配置一半通过initConfig提供。第三方库一通过grunt.loadNpmTask调用。如果
不想一个个手动调用第三方库功能， 可用load-grunt-tasks库自动调用。
4.可以用grunt-contrib-watch这个库监视源文件变化触发自动编译过程。
5.grunt可以注册默认任务， 这样就不需指定任务名字。
6.grunt异步任务通过回调实现结束, 多任务用数组传参。
grunt配置详见gruntFile.js文件
```
##gulp编译项目   编译的项目是pages-boilerplate
```javascript
1.gulp默认配置文件是根目录下的gulpFile.js。gulpFile文件通过导出函数的形式提供
任务供gulp执行。gulp提供src函数获取输入流，提供dist函数提供输出流。通过流的pipe
函数串接处理过程。通过series函数顺序执行多个任务，通过parallel并行执行多个任务。
2.通过gulp-sass、gulp-babel编译样式和脚本，通过gulp-swig编译html模板文件，
通过gulp-imagemin、gulp-uglify压缩文件。
3.通过browserSync启动开发服务器， 监视编译后文件变化自动刷新浏览器；通过gulp提
供的watch函数监视监视源文件触发自动编译过程。通过del库清理上次编译的文件。通过
loadplugins库自动加载第三方plugin。通过useRef合并多个编译后的文件并且解决模板文件中
资源应用路径的问题。

gulp配置详见gulpFile.js
```
